# 408考研网站 - Vue 组件列表文档

本文档说明前端组件的分类、职责和依赖，用于统一使用规范。

## 组件分类原则

组件按照以下原则进行分类重组：

### 基础组件 (basic/)
- **特征**: 高度复用、通用性强、业务耦合度低
- **用途**: 构成基础功能模块，可在多个业务场景中复用
- **设计原则**: 遵循 KISS (保持简洁)、YAGNI (不会需要)、SOLID 原则

### 业务组件 (business/)
- **特征**: 特定业务逻辑绑定、业务专属性强、复用性相对较低
- **用途**: 针对具体业务场景设计，包含特定的业务逻辑
- **设计原则**: 遵循 KISS、YAGNI、SOLID 原则，确保业务逻辑清晰可维护

## 使用约定

- 基础组件通过 `@/components/basic` 按需引入，仅承担通用展示和交互职责。
- 业务组件通过 `@/components/business` 引入，仅在对应业务模块内使用。
- 业务组件不得反向依赖基础组件内部实现，只通过对外暴露的 props 和事件交互。
- 新增组件前优先复用或组合现有基础组件，避免重复造轮子。

## 注意事项

1. **分类清晰**：新增组件时先判断属于基础组件还是业务组件，避免混用。
2. **接口稳定**：基础组件对外暴露的 props / 事件需保持向后兼容。
3. **遵循原则**：组件实现应遵循 KISS、YAGNI、SOLID 原则，避免过度封装。
4. **路径规范**：引用组件时统一使用 `@/components/basic`、`@/components/business` 路径。
5. **文档维护**：新增、删除或重构组件后及时更新本文件。
